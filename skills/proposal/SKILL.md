# 提案書作成スキル (Proposal Creation Skill)

このスキルは、チャットボットやWebシステム開発の提案書を作成する際に、必要な項目を漏れなく網羅し、説得力のあるドキュメントを作成するためのガイドラインとテンプレートを提供します。

## 🚀 提案書に必須の11項目（チェックリスト）

提案書を作成する際は、必ず以下の項目を含めてください。

1.  **企画コンセプト（Concept）**
    -   「一言でいうと何が変わるか？」そのシステム導入の目的と価値を簡潔に表現するキャッチコピー。
    -   例：「家探しの悩みを3分で解決するAIコンシェルジュ」

2.  **導入効果：As-Is / To-Be（Before / After）**
    -   「現状の課題（Before）」と「導入後の理想像（After）」を対比させる。
    -   ユーザー視点とクライアント（企業・管理者）視点の両方を記載する。

3.  **提案システム構成（Architecture）**
    -   使用技術（例：LINE, LIFF, GAS, Dify, GitHub Pages）。
    -   なぜその技術を選んだか（コスト安、保守性、スピード開発など）。

4.  **ユーザー体験フロー（UX Flow）**
    -   ユーザーがどのような画面遷移で操作するか（Step 1〜4）。
    -   Flex Messageやリッチメニューの具体的なイメージ図を見せる。

5.  **管理・運用フロー（Management）**
    -   管理者がどうやってデータを見るか（スプレッドシート連携など）。
    -   運用が簡単であることをアピールする。

6.  **開発・実装範囲（Scope）**
    -   何を作るか（ToDoリスト）。
    -   何を作らないか（スコープ外）も明確にする。

7.  **費用・契約について（Cost & Contract）**
    -   **初期開発費用**: 税抜き価格と税込価格を併記。内訳も記載。
    -   **ランニングコスト**: 月額かかる費用（LINE公式アカウント、OpenAI、サーバー代など）。
    -   誰が契約主体か（クライアント契約か、開発者契約か）を明記。
        -   ※特にOpenAI APIなど従量課金サービスは要注意。

8.  **想定スケジュール（Target Timeline）**
    -   開始からリリースまでの週単位の工程表。
    -   「要件定義」→「開発」→「テスト」→「リリース」。

9.  **共有・ご用意いただきたい事項（Requirements）**
    -   クライアントに用意してもらうものリスト。
    -   例：LINE公式アカウント権限、ロゴ画像、特定サービスのAPIキー。
    -   ※「こちらで用意します」という項目があればそれも明記。

10. **納品物（Deliverables）**
    -   最終的にクライアントに渡すもの。
    -   ソースコード、マニュアル、アカウント情報、ドキュメント類。

11. **免責事項・注意事項（Disclaimer）**
    -   AIのハルシネーション（嘘をつく可能性）。
    -   プラットフォーム（LINE, Google）の仕様変更リスク。
    -   あくまで「予測・診断」であり「保証」ではないこと。

## 📝 テンプレートの使用方法

同ディレクトリにある `template.html` をベースに使用してください。
`[ここ]` で囲まれた部分を、案件に合わせて書き換えてください。

## 💡 提案のコツ（Tips）

-   **専門用語を避ける**: 「LIFF」や「GAS」などの技術用語には必ず簡単な説明（例：「LINEの中で動くアプリ」「Googleの自動化ツール」）を添える。
-   **図解を入れる**: 文字だけでなく、フロー図や表を使って視覚的に分かりやすくする。
-   **「安さ」より「コスパ」を強調**: 単に安いだけでなく、「最新技術を使うから効率的で安い」というロジックで価値を高める。
