# 不動産診断システム 仕様書

本ドキュメントは、マイホーム適正予算診断機能における計算ロジック、データ連携、および判定基準を定義します。
**コードを修正した場合は、必ず本ドキュメントも更新してください。**

## 1. 入力項目
ユーザーからの回答など、計算に使用する入力値です。

| 項目名 | 変数名 | 定義・備考 |
| :--- | :--- | :--- |
| 世帯年収 | `annualIncomeMin` / `annualIncomeMax` | 万円単位。選択肢に応じた範囲で取得（下表参照）。手動入力の場合はmin=max=入力値。 |
| 年齢 | `age` | Q14から取得。選択肢→代表値（20代→25, 30代→35, 40代→45, 50代以上→55）。手動入力の場合はそのまま。未回答時はデフォルト35。 |
| 自己資金（頭金） | `ownCapital` | 万円単位。Q15から取得。「なし」→0、「あり」→入力値（万円）。 |
| 既存借入 | `monthlyDebt` | 円単位。月々の返済合計額。Q5「あり」の場合のみ入力。**返済可能額から差し引いて計算に使用。** |
| 現在家賃 | `currentRent` | 円単位。賃貸の場合のみ入力。安全ラインの月々返済額との差額表示に使用。 |
| 希望借入額 | `desiredBudget` | 万円単位。Q13の回答値（範囲の場合は中央値）。ランク判定に使用。 |

### 1.1. 年収レンジ定義

| 選択肢 | 値 | min（万円） | max（万円） |
| :--- | :--- | :--- | :--- |
| 〜399万 | `LT_400` | 200 | 399 |
| 400〜599万 | `400_600` | 400 | 599 |
| 600〜799万 | `600_800` | 600 | 799 |
| 800〜999万 | `800_1000` | 800 | 999 |
| 1,000万以上 | `GT_1000` | 1000 | 1500 |
| 手動入力 | `MANUAL` | 入力値 | 入力値 |

### 1.2. 年齢選択肢定義

| 選択肢 | 値 | 代表年齢 |
| :--- | :--- | :--- |
| 20代 | `AGE_20S` | 25歳 |
| 30代 | `AGE_30S` | 35歳 |
| 40代 | `AGE_40S` | 45歳 |
| 50代以上 | `AGE_50S` | 55歳 |
| 正確に入力 | `MANUAL` | 入力値 |

## 2. 定数（Config）
Script Properties で設定可能です。未設定時は以下のデフォルト値が適用されます。

| 定数名 | デフォルト値 | 説明 |
| :--- | :--- | :--- |
| `RATE_FLOATING` | **1.0%** | 変動金利（審査金利として使用）。 |
| `RATE_FIXED` | 1.8% | 固定金利（参考用）。 |
| `TERM_YEARS` | 35年 | 返済期間（上限）。年齢により短縮される場合あり。 |

## 3. 返済比率（ハードコード）

### 3.1. 借入可能額（銀行融資目線）＝想定借入可能額の目安
雇用形態・年収に応じた返済比率を適用します。年収はmin/maxそれぞれで判定。

| 条件 | 返済比率 |
| :--- | :--- |
| **公務員** | 40% |
| 年収 400万円以上（公務員以外） | 35% |
| 年収 400万円未満（公務員以外） | 30% |

※ 雇用形態は Q3 の回答値（`PUBLIC`）で判定。

### 3.2. 生活安全ラインの目安（安全ゾーン）
生活を崩さない安全圏として、一律 **20%** を適用します。

## 4. 計算式

年収の下限（min）と上限（max）それぞれで以下を実行し、結果を範囲として出力します。

### 4.1. 返済期間の決定（年齢補正）
完済時年齢80歳を上限とし、返済期間を動的に算出します。
*   `actualYears = min(TERM_YEARS, 80 - age)`
*   最低1年で下限クランプ

例: 50歳の場合 → `min(35, 80-50) = 30年`

### 4.2. 月々返済可能額
*   **上限** = `annualIncome × 10000 × ratioMax / 12 - monthlyDebt`
*   **安全** = `annualIncome × 10000 × 0.20 / 12 - monthlyDebt`

※ 既存借入の月々返済額（`monthlyDebt`）を差し引く。負の場合は0にクランプ。

### 4.3. 借入可能額（PV関数）
元利均等返済方式で算出。
*   `maxLoan = PV(RATE_FLOATING, actualYears×12, maxMonthlyPayment) × tenureFactor`
*   `safeLoan = PV(RATE_FLOATING, actualYears×12, safeMonthlyPayment) × tenureFactor`

### 4.4. 勤続年数補正（tenureFactor）
勤続年数Q4の回答に応じて借入可能額を補正します。

| 勤続年数 | 補正係数 |
| :--- | :--- |
| 1年未満 (`LT_1Y`) | **0.7**（30%減額） |
| それ以外 | 1.0（補正なし） |

### 4.5. 予算総額
*   `maxBudget = floor((maxLoan + ownCapital × 10000) / 10000)` 万円
*   `safeBudget = floor((safeLoan + ownCapital × 10000) / 10000)` 万円

### 4.6. 月々返済目安
上限予算ベースの月々返済額を元利均等返済方式で算出。
*   `月々返済額 = maxLoan × (月利 × (1+月利)^n) / ((1+月利)^n - 1)`

### 4.7. 出力形式
すべて範囲テキストとして生成。minとmaxが同値の場合は単一値で表示。
*   安全予算: `1100〜2300`（万円）
*   上限予算: `1700〜3500`（万円）
*   月々返済: `約5.0〜10.0万円`

## 5. 判定ロジック（ゾーン判定）

ユーザーの「希望購入価格帯 (`desiredBudget`)」と計算結果を比較します。
**保守的判定**（下限年収ベース）を採用。

| ランク | 判定名 | 条件 |
| :--- | :--- | :--- |
| **A** | 安全圏 (Safe) | `desiredBudget` ≤ `safeBudgetMin`（下限年収でも安全） |
| **B** | 検討圏 (Caution) | その他（A, Cいずれにも該当しない） |
| **C** | 警戒圏 (Danger) | `desiredBudget` > `maxBudgetMax`（上限年収でも超過）、または借入不可（年収0等） |

※ `desiredBudget` 不明（0）かつ借入可能な場合→ランクB。

## 6. 出力

### 6.1. LINE Flex Message（ランク別）

ランクごとに以下の内容が変わります。

| セクション | ランクA | ランクB | ランクC |
| :--- | :--- | :--- | :--- |
| ヘッダーカラー | 緑 `#06C755` | オレンジ `#FF9800` | 赤 `#E53935` |
| タイトル | ゆとりのある予算計画です | 標準的な予算計画です | 予算超過の可能性があります |
| アドバイス背景色 | 緑系 | オレンジ系 | 赤系 |

**共通表示項目:**
*   想定借入可能額の目安（範囲）
*   月々の返済目安（範囲）
*   生活安全ラインの目安（範囲）
*   安全ラインの月々返済目安（範囲）
*   現在の家賃との差額（家賃入力時のみ・色分け表示）
*   返済期間注釈（35年未満の場合のみ表示、例: `※返済期間30年（完済時80歳）で計算`）
*   ランク別アドバイス文
*   注釈（借入期間・金利の概算である旨 等）

**CTAボタン（ランク別）:**

| 順位 | ランクA | ランクB | ランクC |
| :--- | :--- | :--- | :--- |
| メイン | ▶ この条件で具体的にプロに相談する | ▶ 安全な進め方をプロに相談する | ▶ 改善ポイントをAIに相談する |
| サブ1 | ↻ 条件を変えて再診断する | ↻ 条件を変えて再診断する | ↻ 条件を変えて再診断する |
| サブ2 | ▶ この条件でAIに詳しく相談する | ▶ この条件でAIに相談する | ▶ 安全な進め方をプロに相談する |

## 9. 診断ID仕様
*   **フォーマット:** `YYYY-MonDD-XXX`
    *   例: `2026-Feb16-K7M`
    *   `YYYY`: 西暦4桁
    *   `Mon`: 月の英略称 (Jan, Feb, ...)
    *   `DD`: 日付2桁
    *   `XXX`: ランダム英数字3桁
*   **目的:** LINE上での電話番号/リンク誤認識を回避するため、英字を含む形式を採用。
