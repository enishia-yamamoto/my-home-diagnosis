# マイホーム診断システム 改善提案書

## 1. 現状の課題：アンケート項目の活用不足

現在の診断ロジックでは、全13問のアンケートのうち **計算に使用しているのは3項目のみ** です。
残りの10項目はスプレッドシートへの保存とDify AI相談時の参考情報にとどまっています。

### 現状の利用状況

| 区分 | 項目 | 計算利用 |
|:---|:---|:---:|
| ✅ 計算に使用 | 世帯年収（Q2） | 借入可能額の算出 |
| | 雇用形態（Q3） | 返済比率の判定 |
| | 希望価格帯（Q13） | ランク判定 |
| ⚠️ 間接利用 | 現在の住まい（Q6） | 家賃差額表示のみ |
| ❌ 未使用 | 購入時期（Q1） | 保存のみ |
| | 勤続年数（Q4） | 保存のみ |
| | 既存借入（Q5） | 保存のみ |
| | 家族構成（Q7） | 保存のみ |
| | 将来の予定（Q8） | 保存のみ |
| | 希望エリア（Q9） | 保存のみ |
| | 物件タイプ（Q10） | 保存のみ |
| | 譲れない条件（Q11） | 保存のみ |
| | 不安なこと（Q12） | 保存のみ |

### 改善案：既存アンケートの計算活用

以下の項目を計算ロジックに組み込むことで、**診断精度を向上**させることができます。

#### ① 既存借入の反映（Q5）
実際の住宅ローン審査では、既存の借入（車ローン・カードローン等）が借入可能額に影響します。

> **改善内容:** 月々の返済可能額から既存借入の返済額を差し引いて計算
> **効果:** 実際の審査結果に近い、より正確な借入可能額を提示

#### ② 勤続年数による補正（Q4）
一般的に勤続年数1年未満の場合、金融機関の審査は厳しくなります。

> **改善内容:** 勤続年数に応じた補正係数（例: 1年未満→借入額70%に制限）を適用
> **効果:** 「審査に通らなかった」というギャップの防止

#### ③ 家族構成・将来予定による安全比率調整（Q7・Q8）
子育て世帯や将来の支出増加が見込まれる場合、安全ラインの基準をより保守的にします。

> **改善内容:** 扶養家族や将来の予定に応じて安全比率を調整（例: 家族3人以上→20%→18%）
> **効果:** ライフプランを考慮した、より実用的なアドバイスが可能

## 2. 将来的な追加機能：エリア別相場のリアルタイム連携

現在の診断では希望エリア（Q9）を収集していますが、**そのエリアの実際の不動産相場との照合は行っていません**。

### 提案：国土交通省 不動産取引価格情報APIの活用

国土交通省が無料で公開している **不動産取引価格情報API** を活用し、ユーザーが選択したエリアの実取引データに基づく相場情報を診断結果に組み込みます。

#### APIの特徴

| 項目 | 内容 |
|:---|:---|
| 提供元 | 国土交通省（公的機関） |
| 費用 | **無料**（APIキー不要） |
| データ内容 | 実際の不動産取引価格、面積、築年数、最寄駅等 |
| 更新頻度 | 四半期ごと |
| 対象 | 全国の住宅・宅地の取引 |

#### 診断結果への組み込みイメージ

現在の診断結果に加えて、以下が表示されます：

```
📍 北海道（戸建て）のエリア相場
   直近の平均取引価格: 約2,800万円
   → あなたの借入可能額の範囲内です ✅
```

#### 期待効果

- 「この予算でこのエリアに住めるのか？」という最も知りたい疑問に直接回答できる
- **公的データ**のため客観性・信頼性が高い
- 運用コスト・メンテナンスコストが極めて低い（自動取得）
- AI相談時にも相場データを参照でき、**より具体的なアドバイス**が可能になる

#### 実装ステップ

| Phase | 内容 | 工数目安 |
|:---|:---|:---|
| Phase 1 | 都道府県×物件タイプの相場取得・Flex Messageへの表示 | 小 |
| Phase 2 | 市区町村レベルの相場表示・予算との適合判定 | 中 |
| Phase 3 | 相場データをDify AIのナレッジベースに格納し、相談品質を向上 | 中 |
